Class: MongoRepo

MongoRepo

new MongoRepo(db, options)

Creates a new MongoRepo on the specified mongo `db`, and whose behavior is controlled by `options`.
Parameters:
Name Type Attributes Default Description
db object The MongoClient instance where the repository gets its collection.
options object Initialization options for the repository.
option.collection string The name of the collection on MongoDb where this repository's data is stored.
option.id string | identityAccessor <optional>
 "_id" Either an optional property name that identifies which of the domain model's properties corresponds to an instance's identity or an identityAccessor callback that returns the model's identity.
option.descriptiveName string <optional>
 "Data model" An optional descriptive name for the domain model; used in logging and error messages.
option.timestampOnCreate Array.<string> <optional>
 [] An optional array of JSON Pointers identifying property paths that the repository will set to the current UTC date when #create() is called.
option.timestampOnUpdate Array.<string> <optional>
 [] An optional array of JSON Pointers identifying property paths that the repository will set to the current UTC date when #update() is called. This repository implementation is one-to-one with a domain model. Its purpose is to provide boilerplate CRUD operations for a single domain model.

_makeUpdateSet(orig, updated) → {object}

Callback method invoked by the repository to calculate the updates to be pushed to the underlying mongodb; enables subclass manipulation of the data right before being stored. The default implementation makes an update set that uses the Mongo Native Client's $set syntax. It constructs an object that contains instructions that send only the differences to the server.

findMatch(match, callback)

Finds objects matching the specified query. This method is a raw pass-through to the underlying driver. Upon success, a stream is returned to the caller via callback. The stream's `data` event must be observed in order to read the resulting raw data that satisfies the query. Streaming is complete when the `end` event is fired.
